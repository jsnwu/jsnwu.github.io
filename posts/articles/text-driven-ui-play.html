Getting started

Mar 30, 2026 #tutorial 2 min

This site is intentionally no-build: static HTML/CSS/JS that GitHub Pages can host.

Where things live

  • /index.html is the homepage.
  • /posts/ holds post pages and the posts index.
  • /posts/posts.json lists posts for the UI.
  • /assets/ has CSS/JS and images.
  • /assets/partials/ holds shared sidebar, topbar (menu, theme, search), and rail HTML; pages load them via data-include in main.js. Edit those files to change chrome site-wide. If you use VS Code Live Server, it can inject reload scripts into .html responses (even inside <svg>), which used to break partials; main.js strips that noise after fetch.

Add a new post

  1. Create /posts/your-slug.html by copying an existing post (keep the data-include placeholders; only change head metadata, breadcrumb, and article body).
  2. Add an entry to /posts/posts.json.
