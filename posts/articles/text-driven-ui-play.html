Text-Driven UI Recorder/Player (ui-play)

Mar 30, 2026 #ui recorder #ui player #text-driven 3 min

Problem

Using AI tool is a daily task for many developers and QAs. However, it is not easy to use Chrome or Playwright MCP to drive UI actions, due to two reasons: high token usage and high latency. It takes long time for the AI to reason the intention of user actions, and need full page content and snapshots in context window. As a result, it's extreamly slow and costly to use AI to drive simple UI interactions.

Solution

ui-play turns plain-text step lines into runnable UI tests. You can debug a flow step-by-step in a small web UI, record stable locators during discovery, and then run established tests fast using page objects only.

The best part is that it doesn't need tokens and it's really fast. It utilizes the locator resolver to quickly try out different locating strategies to find the target element. It supports iframes, distinguish elements by index, and more.

For many UI flows, the hard part isn’t “what to do” — it’s keeping locators stable and making iteration fast. ui-play focuses on a tight loop: write steps in text, run/debug them interactively, record locators once, then execute reliably.

Flow (write → debug → discovery → run)

  • Write: steps are just lines (usually stored in YAML).
  • Debug: interactive runner to run steps one-by-one, add/edit steps, set breakpoints, save.
  • Discovery: executes steps in Playwright and records locators into a page object. If it succeeds, the test becomes “established”.
  • Run: executes established tests using page objects only (fast and less flaky).

Setup

  • Install: npm install
  • Playwright: npx playwright install

Common commands


# Create a test from step lines
npm run uiplay -- init-test my-test "My flow" "Navigate to {{baseUrl}}"

# Record/Update/Debug a test with UI debugger
npm run uiplay:debug my-test

# Discover mode (establish a test, sloidify locators into page object)
npm run uiplay -- discovery my-test

# Run an established test
npm run uiplay -- run my-test

Test case format (YAML)

Test cases live under uiplay/tests/. Variables like {{baseUrl}}, {{env:VAR}}, or per-test variables are substituted at runtime. 

test_case:
  id: my-test
  title: My flow
  established: false
  baseUrl: "{{baseUrl}}"
  env: {}
  variables:
    RUN_ID: "{{timestamp}}"
    RANDOM_SUFFIX: "{{randomStr[6]}}"
  actions:
    - Navigate to {{baseUrl}}
    - Click Sign in button
    - Fill Email address with {{env:PLAYWRIGHT_LOGIN_EMAIL}}
    - Fill Password with {{env:PLAYWRIGHT_LOGIN_PASSWORD}}
    - Click Continue

Step keywords (examples)

  • Navigate to <url>
  • Click <element>
  • Fill <element> with <value>
  • Press <key> (defaults to Enter)
  • Assert visible <element>
  • Pause for N seconds / Pause
